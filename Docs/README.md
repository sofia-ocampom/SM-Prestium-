# SM PRESTIUM

SM Prestium es un software que permite gestionar el préstamo de objetos entre personas de manera organizada, segura y eficiente. El sistema facilita el registro de usuarios, artículos, préstamos, devoluciones y ventas automáticas cuando no se cumplen los tiempos establecidos, ayudando a mejorar el control del inventario y reducir la pérdida de objetos.

# Reporte de vision
 Sm Prestium busca convertirse en una herramienta sencilla, organizada y confiable que permita administrar el préstamo de objetos entre personas de manera eficiente. Está diseñado para controlar múltiples préstamos sin depender de registros manuales o de la memoria.

El sistema permitirá registrar usuarios, artículos, préstamos realizados y devoluciones. También generará certificados de devolución como evidencia del proceso y facturas de venta cuando un objeto no sea entregado dentro del tiempo máximo permitido de 30 días.

El programa contará con un menú en consola amigable e intuitivo, facilitando su uso por personas sin conocimientos técnicos. Además, incluirá funciones administrativas que permitirán consultar estadísticas generales del sistema como el total de préstamos registrados, objetos devueltos, ventas realizadas y usuarios con mayor actividad dentro del sistema.

De esta manera, Sm Prestium permitirá mejorar la organización de los préstamos, optimizar el control del inventario y reducir la probabilidad de pérdida de objetos.
# Especificaciones 
## Funcionales 
El sistema debe permitir registrar usuarios con información básica como:
- Nombre
- Apellido
- Documento
- Correo electrónico
- Tiempo máximo de préstamo permitido
- El sistema debe validar correctamente los datos ingresados por el usuario.
  
 El sistema debe permitir registrar ítems disponibles para préstamo incluyendo: 
- Nombre del objeto
- Categoría
- Precio de compra
- Estado del objeto
- El sistema debe permitir registrar préstamos únicamente a usuarios previamente registrados.
- El sistema debe permitir consultar los préstamos activos organizados por tiempo o fecha.
- El sistema debe permitir registrar devoluciones únicamente cuando existan préstamos activos asociados al usuario.
- El sistema debe generar certificados de devolución automáticamente.
- El sistema debe permitir marcar un objeto como vendido cuando el préstamo supere los 30 días.
- El sistema debe generar automáticamente una factura de venta con el cálculo del valor total incluyendo el impuesto correspondiente.
  
El sistema debe permitir consultar reportes administrativos como:
- Total de préstamos registrados
- Total de objetos devueltos
- Total de ventas realizadas
- Lista de usuarios registrados
- Usuario con mayor número de préstamos
- Usuario con menor número de préstamos
- El sistema debe permitir acceso al módulo administrador mediante usuario y contraseña.
## No funcionales 
Los requisitos no funcionales describen características de calidad del sistema.
- El sistema debe ser fácil de usar para personas sin conocimientos técnicos.
- El sistema debe contar con un menú organizado, claro e intuitivo dentro de la consola.
- El sistema debe responder rápidamente a las acciones del usuario.
- El código debe estar estructurado mediante programación orientada a objetos para facilitar su mantenimiento.
- La información registrada debe almacenarse en archivos planos para permitir su consulta posterior.
- El sistema debe generar mensajes claros y descriptivos para orientar al usuario durante su uso.
- El sistema debe validar correctamente los datos ingresados.
